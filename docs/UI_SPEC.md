## 1. Обзор и дизайн-система

### 1.1 Описание приложения

«ТРІУМФ» — мобильное приложение для управления детской секцией дзюдо. Приложение обслуживает два типа пользователей:

| Роль | Украинское название | Возможности |
|------|---------------------|-------------|
| Тренер | Тренер | Полный доступ: CRUD спортсменов, управление поясами, посещаемость, рейтинг, настройки клуба |
| Родитель/опекун | Батько/опікун | Просмотр команды, рейтинга, привязка своего ребёнка |

> **Бизнес-правило:** первый зарегистрированный пользователь автоматически получает роль «Тренер». Все последующие получают роль «Батько/опікун» и могут быть повышены тренером.

### 1.4 Элементы оформления

- **Border radius:** карточки — 12 dp, кнопки — 8 dp, чипы — 20 dp, аватары — 50% (круг), диалоги — 16 dp
- **Elevation:** карточки — 2, FAB — 6, диалоги — 8
- **Spacing grid:** 4 dp (базовая единица), горизонтальные отступы экрана — 16 dp
- **Icon size:** стандартные — 24 dp, в AppBar — 24 dp, в бейджах — 14 dp

---

### 1.5 Состояния компонентов

Все интерактивные элементы должны иметь визуальные состояния:

- **Default** — нормальное состояние
- **Hovered** (только планшеты) — легкая подсветка
- **Pressed / Ripple** — splash-эффект (Material Design)
- **Focused** — обводка `accent` 2 dp
- **Disabled** — opacity 38%, не интерактивный
- **Loading** — shimmer-анимация или CircularProgressIndicator
- **Error** — красная обводка + иконка предупреждения

**Назначение:** Брендинг и инициализация приложения; проверка сессии авторизации.

### 3.1 Макет

```
┌─────────────────────────────┐
│                             │
│                             │
│                             │
│         [ЛОГОТИП]           │  ← SVG/PNG эмблема дзюдо, 120×120 dp
│       (дзюдо-эмблема)       │
│                             │
│         ТРІУМФ              │  ← displayLarge, Bold, White, letter-spacing 4
│                             │
│  Спортивний клуб · Київ     │  ← bodyMedium, onSurfaceVariant
│                             │
│                             │
│                             │
│       [⟳ Spinner]           │  ← CircularProgressIndicator, accent color
│                             │
└─────────────────────────────┘
```

**Фон:** радиальный градиент от `#1E2C6E` (центр) к `#0F2060` (края)

### 3.2 Поведение

| Шаг | Действие |
|-----|----------|
| 1 | Fade-in логотипа и текста (400 ms) |
| 2 | Проверка Firebase Auth токена (async) |
| 3 | Если авторизован → переход на Main Shell (replace) |
| 4 | Если не авторизован → переход на Login (replace) |
| 5 | Минимальное время показа: 1500 ms |

### 3.3 Компоненты

- **Логотип:** SVG-изображение, центрирован по обеим осям
- **Название:** `ТРІУМФ` — uppercase, широкий межбуквенный интервал
- **Подзаголовок:** `Спортивний клуб · Київ`
- **Спиннер:** `CircularProgressIndicator` цвета `accent`, размер 32 dp

## 4. Поток авторизации

### 4.1 Экран входа

**Route:** `/login`  
**Назначение:** Аутентификация существующего пользователя.

#### Макет

```
┌─────────────────────────────┐
│                             │
│         [ЛОГОТИП]           │  ← 80×80 dp

│   Спортивний клуб · Київ   │  ← bodySmall, onSurfaceVariant
│                             │
│  ┌───────────────────────┐  │
│  │ Email                 │  │  ← TextField, keyboardType=email
│  └───────────────────────┘  │
│                             │
│  ┌───────────────────────┐  │
│  │ Пароль           [👁] │  │  ← TextField, obscureText + toggle
│  └───────────────────────┘  │
│                             │
│  ┌───────────────────────┐  │
│  │       УВІЙТИ          │  │  ← ElevatedButton, primary, full-width
│  └───────────────────────┘  │
│                             │
│  Немає акаунту?             │
│  [Зареєструватися]          │  ← TextButton, accent color
│                             │
└─────────────────────────────┘
```

#### Состояния

| Состояние | Описание | Визуальное поведение |
|-----------|----------|---------------------|
| `default` | Пустые поля | Поля с placeholder, кнопка активна |
| `loading` | Запрос к Firebase | Кнопка показывает `CircularProgressIndicator` внутри, поля заблокированы |
| `error_credentials` | Неверный email/пароль | Красный snackbar «Невірний email або пароль», поля встряхиваются (shake-анимация) |
| `error_network` | Нет соединения | Красный snackbar «Немає підключення до мережі. Перевірте інтернет» |

#### Компоненты

- **Email TextField:**
  - Label: `Email`
  - `keyboardType: EmailAddress`
  - `textInputAction: Next`
  - Иконка-префикс: `email` (onSurfaceVariant)
- **Password TextField:**
  - Label: `Пароль`
  - `obscureText: true` по умолчанию
  - Суффикс: `IconButton` `visibility` / `visibility_off`
  - `textInputAction: Done` → триггер входа
- **Кнопка «УВІЙТИ»:** `ElevatedButton`, ширина 100%, высота 52 dp, `borderRadius: 8`
- **Ссылка «Зареєструватися»:** `TextButton`, цвет `accent`

#### Навигация

| Действие | Переход |
|----------|---------|
| Успешный вход | `Navigator.pushReplacement` → Main Shell |
| Нажатие «Зареєструватися» | `Navigator.push` → RegisterScreen |

| `error_weak_password` | Пароль слабый | Текст ошибки под полем: «Пароль має містити щонайменше 6 символів» |
| `success` | Аккаунт создан | Snackbar «Ласкаво просимо до Тріумфу!», переход на Main Shell |

#### Логика первого пользователя

> Если коллекция `users` пуста в момент регистрации → новый пользователь получает роль `coach`. Всем последующим присваивается роль `parent`.

Визуальная подсказка после успешной регистрации тренера:
- Показывается **модальный диалог** «Ви перший тренер клубу!» с кнопкой «Розпочати роботу»

#### Валидация полей (inline)

| Поле | Правило | Сообщение об ошибке |
|------|---------|---------------------|
| Повне ім'я | Обязательное, мин. 2 символа | «Введіть ваше ім'я» |
| Email | Формат email | «Введіть коректний email» |
| Пароль | Мин. 6 символов | «Пароль має містити щонайменше 6 символів» |

#### Навигация

| Действие | Переход |
|----------|---------|
| Успешная регистрация | `Navigator.pushReplacement` → Main Shell |

#### Состояния экрана

| `filled` | Есть данные | Список карточек |

#### Компонент: ChildCard

```
┌────────────────────────────────────────┐
│  #N   [АВАТАР]   Прізвище Ім'я         │
│  52dp  52dp      bodyLarge, Bold       │
│                  2014 · 45кг · ♂/♀    │  ← bodySmall, onSurfaceVariant
│                  [Жовтий пояс] [★ 23] │  ← BeltBadge + PointsBadge
│                              [✅ badge]│  ← если beltReady=true
└────────────────────────────────────────┘
```

**Спецификация ChildCard:**
- Фон: `surface`
- Border radius: 12 dp
- Padding: 12 dp
- Margin между карточками: 8 dp
- Rank (#N): `titleMedium`, `accent`, ширина 32 dp
- Аватар: 52×52 dp, круглый, инициалы если нет фото, фон `primary`
- Имя: `titleLarge`, Bold, White
- Подстрочная строка: год · вес · иконка пола (`♂` / `♀`)
- BeltBadge: цветная пилюля с названием пояса
- PointsBadge: `★ N`, `accent`
- BeltReady: `✅` появляется в правом верхнем углу карточки

#### Компонент: FilterChips

```
[Пояс ▾]  [Рік н. ▾]  [Тренер ▾]  [Вага ▾]  [Стать ▾]  [До поясу]
```

| Состояние | Вид |
|-----------|-----|
| `inactive` | Фон `surfaceVariant`, текст `onSurfaceVariant` |
| `active` | Фон `accent`, текст `#0F2060` (тёмный), иконка `✓` слева |
| `loading` | Shimmer |

Чипы прокручиваются горизонтально. При нажатии на чип с выпадающим меню (Пояс, Рік н., Тренер, Вага, Стать) — показывается `BottomSheet` с вариантами. Чип «До поясу» — без выпадающего, переключатель (вкл/выкл).

#### FAB

- Видимость: только для роли `coach`
- Иконка: `add`
- Цвет: `accent`
- Действие: переход на AddEditChildScreen (режим добавления)

#### Навигация

| Действие | Переход |
|----------|---------|
| Нажатие на ChildCard | `push` → ChildProfileScreen |
| Нажатие FAB | `push` → AddEditChildScreen (add mode) |

## Приложение C: Адаптивность и доступность

### Безопасные зоны (Safe Areas)

- Учитывать `SafeArea` сверху (status bar) и снизу (home indicator на iOS)
- BottomNavBar: высота 64dp + нижний inset

### Масштаб текста

- Интерфейс должен корректно работать при `textScaleFactor` 1.0–1.3
- Избегать жёстко заданной высоты для компонентов с текстом

### Контрастность

- Соотношение контраста текста на тёмном фоне — минимум 4.5:1 (WCAG AA)
- Белый (#FFFFFF) на `primary` (#1A3A8F): ≈ 8.5:1 ✓
- Accent (#F5A623) на `primary` (#1A3A8F): ≈ 5.2:1 ✓

### Touch Target

- Минимальный размер интерактивного элемента: 48×48 dp
- Чекбоксы и переключатели: `ListTile` с полной шириной

### Тёмная тема

Приложение использует только тёмную тему. Светлый режим не предусмотрен.
