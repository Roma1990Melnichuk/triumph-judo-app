# Judo App — Дизайн та структура проекту

## Концепція

Мобільний застосунок для дзюдо-клубу. Два типи користувачів:
- **Тренер** — повний доступ: управління дітьми, поясами, результатами
- **Батьки** — перегляд списку команди, повний доступ тільки до профілю своєї дитини

---

## Технологічний стек

| Шар | Технологія | Причина |
|-----|-----------|---------|
| UI | Flutter (Dart) | Кросплатформа, Android-ready, Google Play |
| Auth | Firebase Auth | Email/пароль, ролі |
| БД | Firebase Firestore | Офлайн-режим вбудований, до 10к записів легко |
| Фото | Firebase Storage | Аватари дітей |
| Стейт | Riverpod | Реактивний, добре з Firestore |
| Офлайн | Firestore offline persistence | Автоматично |

---

## Ролі та доступ

```
Coach (Тренер)
├── Додавати / редагувати / видаляти дітей
├── Призначати поточний пояс дитині
├── Позначати вправи пояса (склав/не склав)
├── Додавати результати змагань (місце, тип, бали)
├── Редагувати вимоги до поясів (для всіх)
├── Скидати результати (за сезоном)
└── Фільтрувати за тренером (бачать усіх)

Parent (Батько/мати)
├── Переглядати список усіх дітей (тільки ФІО, пояс, місце у рейтингу)
└── Повний профіль — тільки своєї дитини
```

## Структура даних (Firestore)

### Колекція `users`
```
users/{uid}
  email: string
  name: string
  role: 'coach' | 'parent'
  childId: string?        // тільки для батьків — ID їх дитини
```

### Колекція `children`
```
children/{childId}
  firstName: string
  lastName: string
  birthYear: number
  weightCategory: string  // '-25кг', '-30кг', '-34кг', '-38кг', '-42кг', '-46кг', '-50кг', '+50кг'
  currentBelt: BeltLevel  // enum
  photoUrl: string?
  coachId: string
  coachName: string       // денормалізація для швидкого фільтру
  totalPoints: number     // кешований підрахунок для рейтингу
  createdAt: timestamp
```

### Колекція `competition_results`
```
competition_results/{resultId}
  childId: string
  childName: string       // денормалізація
  competitionName: string
  competitionType: string // тип зі списку тренера
  level: 'local' | 'regional' | 'national' | 'international'
  place: 1 | 2 | 3
  points: number          // тренер вводить вручну
  date: timestamp
  seasonYear: number      // для скидання результатів по сезону
  addedByCoachId: string
```

### Колекція `belt_requirements`
```
belt_requirements/{belt}       // документ = рівень пояса
  belt: BeltLevel
  exercises: [
    { id: string, name: string, description: string }
  ]
  updatedAt: timestamp
  updatedByCoachId: string
```

### Колекція `belt_progress`
```
belt_progress/{childId}_{belt}
  childId: string
  belt: BeltLevel
  passed: { [exerciseId]: boolean }
  completedAt: timestamp?    // коли всі вправи здані
```

### Колекція `competition_types`
```
competition_types/{typeId}
  name: string               // тренер додає власні типи
  createdByCoachId: string
```

## Система поясів (дитяче дзюдо)

| # | Пояс | Позначення |
|---|------|-----------|
| 1 | Білий | ⬜ |
| 2 | Біло-жовтий | 🟡 (половина) |
| 3 | Жовтий | 🟡 |
| 4 | Жовто-помаранчевий | 🟠 (половина) |
| 5 | Помаранчевий | 🟠 |
| 6 | Помаранчево-зелений | 🟢 (половина) |
| 7 | Зелений | 🟢 |
| 8 | Зелено-синій | 🔵 (половина) |
| 9 | Синій | 🔵 |
| 10 | Синьо-коричневий | 🟤 (половина) |
| 11 | Коричневий | 🟤 |
| 12 | Чорний (Дан) | ⬛ |

**Логіка:** У профілі дитини показуються вимоги до НАСТУПНОГО пояса.
Тренер може редагувати список вправ для будь-якого пояса — зміни стосуються всіх дітей цього рівня.

### Стартові вимоги (редагуються тренером)

**Білий → Біло-жовтий:**
- Правильне падіння назад (укемі)
- Правильне падіння вбік
- Стійка дзюдоїста (сейза, шизентай)
- Захват (рукав + комір)
- Осото-гарі (підніжка ззовні)
- Оучі-гарі (підніжка зсередини)

**Біло-жовтий → Жовтий:**
- Всі вимоги попереднього рівня
- Укі-госі (кидок через стегно)
- Де-аші-барай (підсічка)
- Хон-кеса-гатаме (утримання)
- Базова боротьба лежачи (неваза)

**Жовтий → Жовто-помаранчевий:**
- Сео-наге (кидок через плече)
- Косото-гарі
- Кокура-мусубі (зав'язування пояса)
- Ката-гурума (базова)
- Рандорі 2 хв

*(і так далі — всі рівні тренер може редагувати в додатку)*

## Навігація та екрани

```
┌─────────────────────────────────────┐
│         Splash / Auth               │
│    Вхід / Реєстрація (тренер)       │
│    Реєстрація батька → пошук дитини │
└──────────────┬──────────────────────┘
               │
       ┌───────┴───────┐
       ▼               ▼
  [ТРЕНЕР]         [БАТЬКО]

ТРЕНЕР (Bottom Nav):
├── 👥 Команда        — список дітей
├── 🏆 Рейтинг        — топ за балами
├── 🥋 Пояси          — вимоги до поясів (редагування)
└── ⚙️  Налаштування  — типи змагань, скидання

БАТЬКО (Bottom Nav):
├── 👥 Команда        — список (тільки ФІО, пояс, рейтинг)
└── 🏆 Рейтинг        — топ за балами
```

### Деталізація екранів

#### 1. Список команди (Тренер)
- Пошук за прізвищем (real-time)
- Фільтри: тренер | рік народження | пояс
- Сортування: за прізвищем (А→Я / Я→А) | за балами
- Картка: фото + ПІБ + пояс + бали + місце в рейтингу
- FAB "+" — додати дитину

#### 2. Профіль дитини (Тренер — повний)
```
┌─────────────────────────────┐
│  [Фото]  Прізвище Ім'я      │
│          2015 р.н. | -38кг  │
│          🟠 Помаранчевий    │
│          Тренер: Іваненко   │
│          🏆 Місце #3 | 42б  │
├─────────────────────────────┤
│  РЕЗУЛЬТАТИ ЗМАГАНЬ         │
│  🥇 Чемпіонат міста 2024    │
│  🥈 Кубок клубу 2024        │
├─────────────────────────────┤
│  ПОТОЧНИЙ ПОЯС: Помаранчевий│
│  ДО НАСТУПНОГО ПОЯСА:       │
│  ✅ Сео-наге                │
│  ✅ Косото-гарі             │
│  ❌ Ката-гурума             │
│  ❌ Рандорі 2 хв            │
└─────────────────────────────┘
[Додати результат] [Редагувати]
```

#### 3. Профіль дитини (Батько — обмежений для чужих)
- Тільки: ФІО, пояс, місце в рейтингу, загальні бали

#### 4. Редактор вимог пояса (тільки тренер)
- Вибір пояса зі списку
- Список вправ з чекбоксами
- Кнопки: додати вправу / видалити / редагувати назву

#### 5. Додати результат змагань
- Вибір дитини (або з профілю)
- Назва змагань / тип (з переліку тренера) / рівень
- Місце: 1, 2, 3
- Бали: ручне введення тренером
- Дата
- Рік сезону (для скидання)

#### 6. Рейтинг
- Топ список за балами
- Фільтр: рік народження | тренер | сезон
- Місце / Фото / ПІБ / Бали / Пояс

#### 7. Налаштування (тренер)
- Керування типами змагань (додати/видалити)
- Скидання результатів (за сезоном — підтвердження)
- Вихід

## Офлайн-режим

Firestore автоматично кешує:
- Список дітей
- Профілі
- Результати змагань
- Вимоги поясів

При відсутності інтернету:
- Перегляд — працює повністю
- Додавання/редагування — ставиться в чергу, синхронізується при з'єднанні

## MVP → Повна версія

### MVP (Phase 1):
- Авторизація (тренер + батько)
- Список дітей з фільтрами
- Профіль дитини
- Результати змагань + рейтинг
- Вимоги до поясів (статичні)

### Phase 2:
- Офлайн-режим (тестування edge cases)
- Редагування вимог поясів тренером
- Фото дітей
- Скидання результатів по сезону

### Phase 3:
- Push-сповіщення батькам (дитина отримала місце)
- Експорт PDF (протокол результатів)
- Google Play публікація
