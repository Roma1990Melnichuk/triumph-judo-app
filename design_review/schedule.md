## Розклад тренувань — ⚠️ Частково відповідає

**Макет:** Явного макету для екрана "Schedule" немає. У дизайн-контексті зазначено: *"SCHEDULE: Not explicitly shown in design. Likely: calendar/week view with training sessions."* Порівняння проводиться відносно загальної дизайн-системи Variant_1 "Premium Martial Arts".

### Що відповідає макету ✅
- Використовується `AppColors.background` як колір фону — відповідає темній палітрі (#050000)
- `AppColors.primary` застосовується для акцентів (іконки, CircleAvatar, кнопка додавання) — відповідає червоному #D50000
- `AppColors.error` для кнопки видалення — коректно
- `AppColors.success` для кнопки "Всі присутні" — функціонально виправдано
- Картки (`Card`) використовуються як поверхні — відповідає surface #1A1A1A
- Українська локалізація всіх текстів — відповідає вимогам

### Що відрізняється від макету ⚠️ / ❌
- **Немає тижневого/місячного calendar view**: дизайн-система передбачає горизонтальний календар (Mon–Sun row), як показано в екрані Journey. Реалізовано лише DatePicker через діалог — відсутній вбудований горизонтальний рядок днів тижня
- **AppBar не відповідає стилю**: у Variant_1 AppBar має градієнтний стиль або відсутній (контент займає весь екран). Поточний AppBar — стандартний Material без кастомного стилю
- **Card не має заокруглення 16–20px**: дизайн вимагає `borderRadius: 16–20`, картки використовують дефолтний радіус Flutter (~12px)
- **Відсутній FAB**: у схожих екранах (Team List) є FAB з "+" іконкою внизу праворуч. Тут кнопка додавання розміщена як `IconButton` у рядку заголовку розділу — менш помітно
- **Відсутній hero/banner блок**: у дизайні ключові екрани мають великий градієнтний hero-card зверху. Екран розкладу починається одразу з date picker без візуального акценту
- **Типографіка заголовку**: `AppBar title` не використовує bold + великий шрифт, як передбачено дизайн-системою (bold headings)
- **Колір тексту**: не скрізь явно вказано `Colors.white` для основного тексту — покладається на тему, що може дати розбіжності
- **`activeThumbColor` у SwitchListTile**: встановлено лише `activeThumbColor`, але `activeTrackColor` не вказано — трек перемикача може не відповідати фірмовому червоному

### Критичні розбіжності
- **Відсутній горизонтальний тижневий календар**: ключовий UI-елемент, характерний для екранів розкладу в дизайн-системі (показано в Journey екрані як Mon–Sun рядок). Поточна реалізація через DatePicker-діалог суттєво знижує зручність та не відповідає очікуваному UX
- **Немає макету від замовника**: без затвердженого wireframe для Schedule неможливо повністю верифікувати відповідність — необхідно отримати конкретний mockup екрана від дизайнера
