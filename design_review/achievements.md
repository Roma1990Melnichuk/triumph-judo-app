## Статистика нагород (Тренер) — ⚠️ Частково відповідає

**Макет:** `Ach.png` (Parent App, VARIANT_12, екран досягнень)

### Що відповідає макету ✅
- Темний фон (`#050000` через `AppColors.background`) — відповідає дизайн-системі
- Групування нагород по категоріях (13 категорій, як у макеті)
- Відображення рідкісності (`rarity.label`) біля кожного бейджа — Звичайні / Рідкісні / Легендарні / Міфічні
- `AchievementIcon` (бейдж-мініатюра) у кожному рядку — відповідає сітці бейджів у макеті
- Кольорова схема акцентів: золотий (`AppColors.accent`) для лічильника нагород, червоний (`AppColors.primary`) для статистики
- Кутовий радіус карток та чіпів: 12px — відповідає дизайн-системі
- Bottom sheet з деталями нагороди: бейдж + назва + рідкісність + опис + лічильник — відповідає очікуваному патерну

### Що відрізняється від макету ⚠️ / ❌
- **Призначення екрана розходиться з макетом:** `Ach.png` показує список нагород з точки зору батька/дитини (64 нагороди, що дитина може отримати), тоді як реалізований екран — це адміністративна статистика для тренера ("скільки разів видавалась кожна нагорода"). Це різні екрани з різними UX-завданнями.
- **Відсутній загальний лічильник "64 всього / 13 категорій"** — у макеті показано загальну кількість нагород та категорій як заголовок.
- **Відсутня сітка (grid layout)** — макет `Ach.png` показує бейджі у сітковому вигляді (grid), а не у списку (`ListView` з `ListTile`). Реалізовано лінійний список рядків.
- **Статистичні чіпи у верхній частині ("Видано / Спортсменів / Не видавались")** відсутні у макеті — це додаткова функціональність тренера, не передбачена дизайном.
- **Фільтр "Всі / Не видавались"** відсутній у макеті — макет не показує такого фільтра.
- **AppBar заголовок "Статистика нагород"** — у макеті відсутній такий заголовок; макет показує просто список нагород без appbar-заголовка з адміністративним контекстом.
- **Лічильник кількості видань** (`earned.length`) у трейлінгу кожного рядка відсутній у макеті — макет показує лише бейдж + назву + рідкісність без кількісного показника.
- **Bottom sheet зі списком спортсменів** не передбачений у макеті `Ach.png`.

### Критичні розбіжності
- **Концептуальна розбіжність:** Реалізований екран є адміністративним інструментом тренера для перегляду статистики видачі нагород, тоді як макет `Ach.png` описує екран батьківського/дитячого застосунку з повним каталогом доступних нагород. Необхідно уточнити у замовника, чи потрібен окремий макет для адміністративної статистики тренера, або чи цей екран має повторювати каталог нагород з `Ach.png`.
- **Grid vs List:** Макет явно використовує сіткове відображення бейджів — це принципова відмінність від реалізованого `ListView`, яка суттєво впливає на зовнішній вигляд екрана.
